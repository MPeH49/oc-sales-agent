Why Cowork Projects over regular Projects?

Both can work, but Cowork gives you four things regular Projects can't — and they all matter for a sales agent.

◉ Scoped memory
REGULAR PROJECT
No memory between chats
COWORK PROJECT
Remembers across sessions within each project
→ The agent learns your preferences, remembers past prospects, notices patterns over time
◎ Scheduled tasks
REGULAR PROJECT
Manual only — you start every conversation
COWORK PROJECT
Auto-runs on a daily/weekly cadence
→ Market research, follow-up reminders, and competitor monitoring run without you lifting a finger
◈ Local file access
REGULAR PROJECT
Upload files per chat (200k context limit)
COWORK PROJECT
Reads/writes to a local folder — persistent
→ Estate profiles, the Foresight 68 list, outreach logs — always available, always up to date
◆ Execution
REGULAR PROJECT
Advises you — you do the work
COWORK PROJECT
Creates files, builds docs, updates logs directly
→ It doesn't just suggest an email — it drafts it, saves it, and logs that it was sent
One caveat
Cowork Projects are local to your machine — you can't share them with Rich directly (yet). You'd build and run the agent, then share outputs. If Rich needs to use a module himself, set that one up as a regular browser-based Claude Project.
